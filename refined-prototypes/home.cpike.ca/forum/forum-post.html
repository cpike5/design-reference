I've been working with Go for a few years now, and I've seen a lot of different approaches to handling concurrent operations. With the introduction of more async patterns, I wanted to start a discussion about best practices.

In particular, I'm interested in:

  • Proper goroutine management and cleanup
  • Channel patterns for communication
  • Context timeouts and cancellation
  • Error handling in concurrent code

What practices have you found most effective in your Go projects? I'd love to hear from the community!

Great question! I've found that using context for cancellation is absolutely critical. Here's what I typically do:

func handleRequest(ctx context.Context, req *Request) error {
    // Set timeout for the entire operation
    ctx, cancel := context.WithTimeout(ctx, 30*time.Second)
    defer cancel()

    // Goroutines check the context
    resultCh := make(chan Result)
    go func() {
        select {
        case <-ctx.Done():
            return
        case resultCh <- processRequest(req):
        }
    }()

    select {
    case <-ctx.Done():
        return ctx.Err()
    case result := <-resultCh:
        return handleResult(result)
    }
}

The key is always using context to coordinate goroutine lifecycle. Never rely on goroutines just "ending naturally".

Excellent points from CodeWizard. I'd also emphasize the importance of buffering your channels strategically. Unbuffered channels are great for synchronization but can lead to deadlocks if you're not careful.

For worker pools, I recommend using a semaphore pattern:

type Semaphore chan struct{}

func NewSemaphore(n int) Semaphore {
    return make(Semaphore, n)
}

func (s Semaphore) Acquire() { s <- struct{}{} }
func (s Semaphore) Release() { <-s }

This gives you fine-grained control over goroutine concurrency.
